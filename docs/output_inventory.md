# Output Inventory - Literature Meta-Analysis

Paths are relative to `projects/templates/template_literature_meta_analysis/`. Every file under `output/` is disposable and regenerated from source/config/fixtures.

| File | Role | Generated by |
| --- | --- | --- |
| `output/data/corpus.jsonl` | Deduplicated literature corpus | `scripts/01_literature_search.py` or fixture regeneration path |
| `output/data/subfield_classification.json` | Per-subfield counts | `src/analysis/pipeline_runner.py` via `scripts/02_meta_analysis_pipeline.py` |
| `output/data/subfield_timeline.json` | Year-by-subfield matrix | `src/analysis/pipeline_runner.py` |
| `output/data/temporal_analysis.json` | Publication growth metrics | `src/analysis/temporal_analysis.py` |
| `output/data/tfidf_data.json` | TF-IDF matrix and labels | `src/analysis/text_processing.py` / pipeline runner |
| `output/data/topics.json` | NMF topic summaries | `src/analysis/topic_modeling.py` |
| `output/data/citation_network.json` | Citation graph metrics | `src/analysis/citation_network.py` |
| `output/data/citation_graph.gml` | Citation graph exchange file | `src/analysis/citation_network.py` |
| `output/data/nanopublications.jsonl` | Optional extracted assertions | `src/knowledge_graph/kg_runner.py` |
| `output/data/nanopublications.trig` | RDF/TriG nanopublications | `src/knowledge_graph/nanopublication.py` |
| `output/data/hypothesis_scores.json` | Citation-weighted hypothesis scores | `src/knowledge_graph/hypothesis.py` |
| `output/data/hypothesis_trends.json` | Evidence-over-time series | `src/knowledge_graph/hypothesis.py` |
| `output/data/assertion_summary.json` | Assertion type and hypothesis counts | `src/knowledge_graph/kg_runner.py` |
| `output/figures/*.png` | Publication figures | `src/visualization/figure_runner.py` via `scripts/04_generate_figures.py` |
| `output/figures/figure_registry.json` | Figure registry for downstream docs/rendering | `src/visualization/figure_runner.py` |
| `output/manuscript/*.md` | Token-resolved manuscript sources | `src/manuscript/variables.py` via `scripts/05_inject_variables.py` |
| `output/fulltext/` | Optional full-text availability/download artifacts | `scripts/06_fulltext_assessment.py` |

When adding an artifact, update this table, add or update tests for the producer, and document whether the manuscript consumes it.
