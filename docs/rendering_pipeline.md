# Rendering Pipeline

The literature exemplar renders by hydrating manuscript variables from generated JSON artifacts, then delegating PDF/HTML/DOCX/EPUB work to the root template renderers.

## Phase 1: Produce Data Artifacts

```bash
uv run python projects/templates/template_literature_meta_analysis/scripts/02_meta_analysis_pipeline.py
uv run python projects/templates/template_literature_meta_analysis/scripts/03_build_knowledge_graph.py --max-papers 0
uv run python projects/templates/template_literature_meta_analysis/scripts/04_generate_figures.py --dpi 300
```

Use `01_literature_search.py` before Phase 1 only when intentionally refreshing the corpus from live engines.

## Phase 2: Hydrate Manuscript Tokens

```bash
uv run python projects/templates/template_literature_meta_analysis/scripts/05_inject_variables.py
```

Inputs are `manuscript/config.yaml` plus `output/data/*.json` / `output/data/*.jsonl`. Outputs are resolved markdown files under `output/manuscript/`.

## Phase 3: Validate Markdown Before Render

```bash
uv run python -m infrastructure.validation.cli prerender   projects/templates/template_literature_meta_analysis/manuscript --repo-root .
```

## Phase 4: Render

```bash
uv run python scripts/03_render_pdf.py --project templates/template_literature_meta_analysis
```

Working render artifacts land under `projects/templates/template_literature_meta_analysis/output/`. Repo-level copied outputs are generated by:

```bash
uv run python scripts/05_copy_outputs.py --project templates/template_literature_meta_analysis
```

## Common Failures

- Literal `{{TOKEN}}`: rerun `scripts/05_inject_variables.py`; if still present, add the token in `src/manuscript/variables.py` and test it.
- Missing figure: rerun `scripts/04_generate_figures.py --dpi 300` and check `output/figures/figure_registry.json`.
- Citation error: check `manuscript/references.bib` and the generated `_combined_manuscript.log`.
- Mermaid/Chrome error: install `chrome-headless-shell` as described in the root rendering docs, then rerun `scripts/03_render_pdf.py`.
